# Планировщик жизни / Life Planner

PWA-приложение: **траты** (с конвертацией по курсу НБРБ), **задачи по дому**, **покупки**
и **календарь** в стиле GitHub-contributions. Работает на ПК и Android, без своего сервера —
данные синхронизируются через приватный GitHub-репозиторий.

## Возможности
- 💸 **Траты** — категории, бюджеты, конвертация BYN / USD / RUB по официальному курсу
  Нацбанка РБ (`api.nbrb.by`).
- 🏠 **Задачи по дому** — приоритеты, повторяемость, сроки, фильтры.
- 🛒 **Покупки** — несколько списков, количество, цены, итоги.
- 📅 **Календарь** — микро-задачи по дням и heatmap «зелёных квадратов» как в Git.
- 🌗 Светлая / тёмная / системная тема, 🌐 русский / английский.
- ☁️ Синхронизация ПК ↔ телефон через GitHub (offline-friendly, PWA).

## Стек
React 19 · TypeScript · Vite · Tailwind v4 · zustand · react-i18next · vite-plugin-pwa.

## Синхронизация через GitHub

Приложение хранит все данные в одном файле `data.json` в **приватном** репозитории
и читает/пишет его напрямую через GitHub Contents API. Сервер не нужен.

### 1. Создать приватный репозиторий для данных
1. github.com → **New repository** → имя, например `planner-data`.
2. Visibility: **Private**.
3. Отметить **Add a README** (чтобы появилась ветка по умолчанию).
4. Create repository.

### 2. Создать токен (fine-grained PAT)
1. github.com → Settings → Developer settings → **Personal access tokens → Fine-grained tokens** → *Generate new token*.
2. **Repository access** → *Only select repositories* → выбрать `planner-data`.
3. **Permissions** → Repository permissions → **Contents: Read and write**.
4. Срок — на усмотрение → *Generate token* → **скопировать** (показывается один раз).

### 3. Подключить в приложении
Открыть **Настройки → Синхронизация (GitHub)**, ввести:
- Владелец — твой логин GitHub
- Репозиторий — `planner-data`
- Путь — `data.json` (по умолчанию)
- Ветка — `main`
- Токен — вставить PAT

Нажать **Подключить**. Дальше каждое изменение автоматически уходит в репозиторий
(debounce ~2.5 c), а при открытии/возврате фокуса и раз в минуту приложение подтягивает
свежую версию. На втором устройстве вводишь те же данные — синхронизация заработает.

> 🔐 **Безопасность.** Токен хранится только в `localStorage` твоих устройств и **не**
> попадает в `data.json`. Используй fine-grained токен с доступом только к этому одному
> репозиторию — тогда при утечке под угрозой лишь данные планировщика.

> ⚠️ **Слияние.** Стратегия last-write-wins по времени изменения на уровне документа:
> при одновременной правке оффлайн на двух устройствах победит более поздняя версия целиком.
> При обычном использовании (синхронизация частая) конфликтов не возникает.

## Установка как приложение (PWA)
- **Android (Chrome):** меню ⋮ → «Установить приложение» / «На главный экран».
- **ПК (Chrome/Edge):** значок установки в адресной строке.

После установки работает оффлайн; синхронизация догонит при появлении сети.

## Хостинг (чтобы открывать с телефона и ПК)
Варианты — на выбор (решается отдельно):
- **GitHub Pages** — бесплатно, HTTPS. Собрать `npm run build`, опубликовать `dist/`.
- **Локально из сети** — `npm run preview -- --host`, открывать по IP машины
  (для PWA-установки понадобится HTTPS).
