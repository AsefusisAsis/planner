# Планировщик жизни / Life Planner

Личный планировщик-PWA: финансы, задачи, покупки, календарь, здоровье, банковские карты —
в одном приложении. Работает на ПК и телефоне, **без своего сервера**: данные синхронизируются
через приватный GitHub-репозиторий.

**Живая версия:** https://asefusisasis.github.io/planner/

## Возможности
- 🏠 **Главная** — живые часы, курсы валют (НБРБ), блок «Требует внимания» (просрочки, превышение
  бюджета, ближайший платёж, события), настраиваемые виджеты + быстрый ввод.
- 💰 **Финансы** — доходы/расходы и баланс, категории с бюджетами, кольцевая диаграмма, поиск и
  фильтры, повторяющиеся платежи, тренд за 6 месяцев, мультивалюта **BYN / USD / RUB / EUR** по курсу НБРБ.
- ✅ **Дом (задачи)** — приоритеты, повторяемость, сроки, описание и подзадачи-шаги, группировка
  (просрочено / сегодня / ближайшие).
- 🛒 **Покупки** — списки, количество и цены, прогресс, итоги, «→ в траты», частые товары.
- 📅 **Календарь** — месячный вид с событиями по времени + режим «Активность» (heatmap как в GitHub).
- ❤️ **Здоровье** — нормы калорий/БЖУ/воды/ИМТ (Миффлин-Сан Жеор), дневник веса и замеры, пример
  меню, дневник еды (база продуктов + способ приготовления меняет КБЖУ + трекер воды), тренировки
  (план под инвентарь + журнал с heatmap).
- 💳 **Карты** — банковские (бренд, маскирование, копирование, опц. шифрование под мастер-пароль) и
  скидочные (штрихкод Code128 + полноэкранный показ для сканера).
- ⚙️ **Настройки** — тема (светлая/тёмная/системная), язык (RU/EN), базовая валюта, синхронизация,
  резервная копия (экспорт/импорт), курсы.
- ☁️ Оффлайн-friendly PWA + синхронизация ПК ↔ телефон через GitHub.

## Стек
React 19 · TypeScript (strict) · Vite · Tailwind v4 · zustand · react-i18next · vite-plugin-pwa · Vitest.

## Запуск и разработка
```bash
npm install
npm run dev       # дев-сервер (обычно http://localhost:5173)
npm run build     # прод-сборка в dist/
npm run preview   # предпросмотр прод-сборки
npm test          # тесты (Vitest): слияние, расчёты здоровья, конвертация валют
```

### Структура
```
src/
  modules/        # экраны: dashboard, expenses, home, shopping, calendar, health, cards, settings
  components/     # общие: ui, Heatmap, Donut, Barcode, LineChart, SearchModal, Layout
  services/       # github.ts (Contents API), merge.ts (3-way merge), nbrb.ts (курсы)
  store/          # zustand: данные + синхронизация
  lib/            # localConfig (GitHub-креды/мета), id, theme
  types.ts        # модель данных (AppData) — то, что лежит в data.json
```

## Синхронизация через GitHub (как настроить)

Все данные хранятся в одном файле `data.json` в **приватном** репозитории; приложение читает/пишет
его напрямую через GitHub Contents API. Каждому пользователю нужен свой репозиторий и токен.

**1. Приватный репозиторий для данных**
github.com → **New repository** → имя напр. `planner-data` → **Private** → отметить **Add a README** → Create.

**2. Токен (fine-grained PAT)**
github.com → Settings → Developer settings → **Personal access tokens → Fine-grained tokens** → Generate.
- **Repository access** → Only select repositories → `planner-data`
- **Permissions** → Repository permissions → **Contents: Read and write**
- Generate → **скопировать** (показывается один раз).

**3. Подключить в приложении**
**Настройки → Синхронизация (GitHub)**: владелец (логин GitHub), репозиторий (`planner-data`),
путь `data.json`, ветка `main`, токен. → **Подключить**. На втором устройстве — те же данные.

> 🔐 Токен хранится только в `localStorage` устройства и **не** попадает в `data.json`. Используй
> токен с доступом только к этому одному репозиторию.
>
> 🔁 Синхронизация: при изменении (debounce) и периодически приложение делает **pull → слияние по
> записям (3-way merge, `services/merge.ts`) → push только если данные реально изменились**.
> Удаления и одновременные правки с разных устройств учитываются корректно.

## Деплой (GitHub Pages)
Код в публичном репо `AsefusisAsis/planner`. Push в `main` → автоматическая сборка и публикация через
`.github/workflows/deploy.yml` (GitHub Actions). Базовый путь подставляется из `VITE_BASE=/<repo>/`,
роутинг — `HashRouter` (без 404 на Pages).

Первый раз Pages включается вручную: **Settings → Pages → Build and deployment → Source: GitHub Actions**.

## Установка как приложение (PWA)
- **Android (Chrome):** меню ⋮ → «Установить приложение».
- **ПК (Chrome/Edge):** значок установки в адресной строке.

После установки работает оффлайн; синхронизация догонит при появлении сети.
